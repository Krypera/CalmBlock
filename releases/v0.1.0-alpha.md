# CalmBlock v0.1.0-alpha (Draft)

This draft is prepared for the first public alpha release.  
It is intentionally conservative and aligned with CalmBlock's current scope.

## Highlights

- Privacy-first architecture with no telemetry, no analytics, no remote code.
- DNR-first blocking model with four default groups:
  - Ads
  - Trackers
  - Annoyances
  - Strict Privacy
- Fast popup controls for global toggle and per-site pause/protect.
- Local-only settings model with import/export.

## Engineering and Quality

- TypeScript strict mode enforced.
- Unit/content/integration-style tests included.
- CI workflow runs on push and PR:
  - `npm ci`
  - `npm run typecheck`
  - `npm test`
  - `npm run build`

## Open-Source Readiness

- Contributor onboarding docs and templates are in place.
- Security policy and issue/PR templates are included.
- README and project docs are aligned with current behavior and limitations.

## Known Limitations

- Not intended as full parity with mature maximalist blockers.
- DNR constraints limit advanced rule syntax support.
- Strict mode may break some sites.
- Rules are curated MVP baseline and still evolving.

## Privacy Guarantees (Unchanged)

- No telemetry
- No analytics SDKs
- No account system
- No cloud sync
- No remote code execution

## Upgrade Notes

- Legacy storage keys (`quietblock.*`) are migrated to `calmblock.*` on startup.
- Existing users should retain settings after migration.
